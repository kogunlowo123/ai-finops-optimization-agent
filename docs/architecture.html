ai-finops-optimization-agent

Multi-cloud FinOps agent for cost analysis, anomaly detection, rightsizing, and optimization

Python 3.11+ AI/ML v0.1.0

Executive Summary

A multi-cloud FinOps agent that analyzes costs across AWS, Azure, and GCP using cloud-native billing APIs and LLM-powered anomaly detection. The agent identifies optimization opportunities including instance rightsizing based on CPU utilization, reserved instance and savings plan recommendations, and waste elimination for idle/orphaned resources. It includes safe execution capabilities with dry-run support for resource cleanup, start/stop scheduling for non-production workloads, and reservation purchase workflows with approval gates. Reporting is delivered via Slack webhooks, HTML email, and JSON dashboard exports.

Data Flow / Request Flow

  1. Cost Collection -- Cloud-specific collectors pull cost data from AWS Cost Explorer, Azure Cost Management, and GCP BigQuery billing exports. Utilization metrics come from CloudWatch, Azure Monitor, and GCP Monitoring.
  2. Anomaly Detection -- Historical cost data is analyzed using Z-score statistical methods. Anomalies exceeding the configurable threshold (default 20%) are flagged and explained via LLM.
  3. Rightsizing Analysis -- CPU utilization data (default 7-day window) is compared against instance capacity. Resources below the idle threshold (default 5%) are flagged. Recommendations include target instance type with headroom buffer (default 20%).
  4. Waste Detection -- Unattached EBS volumes, unused Elastic IPs, idle instances, and orphaned resources are identified across all enabled providers.
  5. RI/SP Advisory -- Consistent usage patterns are analyzed to recommend reserved instance or savings plan purchases with projected monthly savings.
  6. Optimization Execution -- Scheduler applies start/stop rules for tagged non-production resources. Cleanup removes waste with dry-run safety. Reservation purchaser previews before committing.
  7. Reporting -- Results are compiled into dashboard JSON, Slack summaries, and HTML email reports with per-resource savings breakdowns.

Industry Adaptation

Healthcare

HIPAA workload cost tracking. Separate cost analysis for PHI-processing infrastructure. Compliance-aware scheduling (no downtime for critical patient systems).

Finance

Trading workload cost optimization with low-latency constraints. PCI-DSS scope resource tracking. Regulatory compute reservation for burst capacity during market hours.

Government

FedRAMP/GovCloud cost analysis. IL4/IL5 workload cost segregation. Mandatory reserved capacity for continuity of operations (COOP) requirements.

Retail

Seasonal scaling cost forecasting. Black Friday/holiday burst capacity planning. Cost allocation by storefront, fulfillment, and analytics workloads.

SaaS

Per-tenant cost attribution. Multi-tier (free/pro/enterprise) infrastructure cost modeling. Usage-based pricing alignment with actual cloud spend.

Production Readiness Checklist

  • Configure cloud credentials for each enabled provider (AWS/Azure/GCP)
  • Validate read-only access with a dry-run analysis
  • Set OPENAI_API_KEY for anomaly explanations
  • Tune anomaly threshold, idle CPU threshold, and rightsizing headroom
  • Run initial analysis and review recommendations before enabling executors
  • Test ResourceCleanup with dry_run=True
  • Configure Slack webhook and/or SMTP for automated reporting
  • Set up cron/scheduled task for daily cost reports
  • Tag non-production resources for scheduler rules
  • Review RI/SP recommendations with finance team before enabling purchase

Deployment

Local Development

git clone https://github.com/your-org/ai-finops-optimization-agent.git
cd ai-finops-optimization-agent
python -m venv .venv
source .venv/bin/activate
pip install -e ".[all-clouds,dev]"
cp .env.example .env
python -m examples.daily_report

Production

# Install with specific cloud support:
pip install -e ".[aws]"        # AWS only
pip install -e ".[azure]"      # Azure only
pip install -e ".[all-clouds]" # All providers

# Schedule daily reports via cron:
# 0 8 * * * /path/to/.venv/bin/python -m examples.daily_report
